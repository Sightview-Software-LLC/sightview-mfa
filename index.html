MFA Service API Documentation

Overview

The MFA Service provides two sets of APIs:

  1. MFA Session API - Core MFA functionality for initiating and confirming multi-factor authentication
  2. Admin API - Administrative endpoints for managing users, notices, and access logs

Authentication

X-Api-Key Authentication

Used for MFA Session API endpoints. The API key must be included in the request header:

X-Api-Key: <your-system-secret>

The key must match a configured system secret in the MFA service's SystemList configuration.

Azure AD Authentication

Used for Admin API endpoints. Requires a valid Bearer token from Azure AD:

Authorization: Bearer <token>

POST /api/mfasession/start

Initiates a new MFA session for a user. Returns a URL where the user should be redirected to complete MFA.

Authentication

X-Api-Key header required

Parameters

Request Body

{
  "email": "string // User's email address",
  "userId": "string // Unique user identifier",
  "systemName": "string // Calling system name",
  "command": "string // Command type: Auth, Register, ResetPw, or Manage"
}
Field Type Required Description
email string Yes User's email address
userId string Yes Unique identifier for the user in the calling system
systemName string Yes Name of the calling system (must match configuration)
command enum No MFA command type. Defaults to appropriate value based on user state

Command Values:

  • Auth - Standard authentication flow
  • Register - New user registration/enrollment
  • ResetPw - Password reset flow
  • Manage - Manage MFA settings

Response

Success Response

200 OK 
/mfa/enrollment/init?sessionId=12345678-1234-1234-1234-123456789012

Returns a relative URL path where the user should be redirected. The actual URL depends on:

  • New users or users without MFA: /enrollment/init
  • Existing users with MFA: /auth
  • ProxyPrefix is prepended if configured for the system

Error Responses

  • 400 Bad Request - Missing required fields or database error
  • 401 Unauthorized - Invalid or missing API key

Example Request

POST https://mfaservice.local/api/mfasession/start
X-Api-Key: <your-api-key>
Content-Type: application/json

{
  "email": "user@example.com",
  "userId": "usr_123456",
  "systemName": "PHPPortal",
  "command": "Auth"
}

Example Response

/mfa/auth?sessionId=a7b8c9d0-1234-5678-9012-e3f4567890ab

  "userId": "string // Unique user identifier",
  "systemName": "string // Calling system name"
}
Field Type Required Description
email string Yes User's email address
userId string Yes Unique identifier for the user
systemName string Yes Name of the calling system

Response

Success Response

200 OK 
{
  "userId": "usr_123456",
  "sessionId": "a7b8c9d0-1234-5678-9012-e3f4567890ab",
  "mfaEnabled": true,
  "preferredMfa": "totp",
  "lastMfaValidation": "2024-05-03T15:45:23Z",
  "cellNumber": "+1234567890",
  "sessionExpiry": "2024-05-03T16:00:23Z"
}
Field Type Description
userId string The user ID that was passed in
sessionId Guid The MFA session identifier
mfaEnabled boolean Whether MFA is enabled for this user
preferredMfa string User's preferred MFA method (totp, sms, email)
lastMfaValidation DateTime ISO 8601 timestamp of last successful MFA
cellNumber string User's phone number (may be null)
sessionExpiry DateTime When the MFA session expires

Error Responses

  • 400 Bad Request - Missing required fields
  • 401 Unauthorized - Invalid API key or user not found

}
